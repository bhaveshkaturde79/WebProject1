1. What is SQL Injection?

SQL Injection (SQLi) is a type of cyber attack where an attacker inserts malicious SQL code into input fields like login forms or search boxes to manipulate a database.

It happens when user input is not properly validated or sanitized.

2. How SQL Injection Works

Example of vulnerable login query:

SELECT * FROM users 
WHERE username = 'admin' AND password = '1234';

If attacker enters this in password field: 

' OR '1'='1

The query becomes: 

SELECT * FROM users 
WHERE username = 'admin' AND password = '' OR '1'='1';

Since '1'='1' is always true, attacker bypasses login.

3. Types of SQL Injection

  • Authentication Bypass – Bypass login system
  • Union-Based SQLi – Extract data using UNION query
  • Blind SQL Injection – Get information without visible errors
  • Error-Based SQLi – Use database error messages

4. Real-World Impact

SQL Injection can lead to:

  • Data theft (Passwords, Credit cards)
  • Data deletion
  • Full database access
  • Website takeover

5. How to Prevent SQL Injection

  • Use Prepared Statements (Parameterized Queries)
  • Use ORM frameworks
  • Validate and sanitize input
  • Use Web Application Firewall (WAF)
  • Limit database permissions
